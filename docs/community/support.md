## Getting Help

### GitHub Discussions

For questions and discussions:

[github.com/microsoft/amplifier/discussions](https://github.com/microsoft/amplifier/discussions)

Good for:

- Usage questions
- Feature discussions
- Sharing use cases
- Community support

### GitHub Issues

For bugs and feature requests:

[github.com/microsoft/amplifier/issues](https://github.com/microsoft/amplifier/issues)

Good for:

- Bug reports (with reproduction steps)
- Feature requests
- Documentation issues

## Reporting Bugs

When reporting bugs, include:

1. **Amplifier version**: `amplifier --version`
2. **Python version**: `python --version`
3. **Operating system**
4. **Steps to reproduce**
5. **Expected behavior**
6. **Actual behavior**
7. **Error messages** (full stack trace if available)

### Example Bug Report

```markdown
## Description
Tool execution fails silently when...

## Environment
- Amplifier: 1.0.0
- Python: 3.11.5
- OS: macOS 14.0

## Steps to Reproduce
1. Run `amplifier run --profile dev "..."`
2. Observe...

## Expected Behavior
Should...

## Actual Behavior
Instead...

## Error Output
```
[paste error here]
```
```

## Security Issues

For security vulnerabilities, please follow responsible disclosure:

**DO NOT** create public issues for security problems.

Instead, report via:
- Microsoft Security Response Center
- See [SECURITY.md](https://github.com/microsoft/amplifier/blob/next/SECURITY.md)

## Commercial Support

Amplifier is an open source project. For commercial support inquiries, please contact Microsoft through official channels.

## Community Guidelines

When seeking support:

- Be respectful and patient
- Provide context and details
- Search before asking
- Help others when you can

See our [Code of Conduct](https://opensource.microsoft.com/codeofconduct/).
